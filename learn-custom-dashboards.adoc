= 了解自定义控制面板
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
作为云和存储管理员，自定义仪表板可让您清楚地查看所管理的系统和工作负载。它们允许您构建和个性化与您的工作、优先事项和业务目标相匹配的视图，而不是局限于预设的布局。

自定义仪表板使用 AI 创建 Workload Factory 环境的个性化视图。您只需用简单的语言描述要监控的内容，系统就会根据您的需求自动构建可视化图表。

Workload Factory 帐户中的每个人都可以查看所有自定义仪表板。此共享设置可帮助团队在整个组织中进行协作并查看相同的运营信息。



[[why-use-custom-dashboards]]
== 为何使用自定义控制面板

Workload Factory 用户管理许多工作负载、环境和优先级，但标准仪表板仅显示预设视图。自定义仪表板有助于解决几个运营挑战：

* *快速回答关键问题*：专注于您理解中最重要的问题或差距，而不是一般的系统统计数据。
* *立即监控重要指标*：跟踪与您当前优先事项相匹配的指标，而不是一组固定的运行状况指标。
* *将技术数据与业务结果联系起来*：展示系统性能如何影响运营和业务目标。
* *避免在工具之间切换*：在一个地方查看所有相关信息，无需在工具、页面或导出之间切换。




[[what-you-can-do-with-custom-dashboards]]
== 自定义控制面板的功能

在设置过程中，AI 引导对话框可帮助您定义：

* 要监控的工作负载和资源
* 对您的工作至关重要的关键指标
* 要比较的时间范围和期间
* 按地区、工作负载或其他类别查看
* 需要跟踪和显示的规则或要求




[[considerations-and-limitations]]
== 注意事项和限制



[[aws-credentials-and-permissions]]
=== AWS 凭据和权限

要创建或查看自定义仪表板，您需要具备所监控资源的正确 AWS IAM 权限。确保 Workload Factory 帐户对仪表板使用的数据源具有读取访问权限。

为了构建仪表板，Workload Factory 使用只读 AWS 凭据。这些凭据仅允许对以下服务执行读取操作，例如 `fsx:DescribeVolumes`和 `fsx:DescribeFileSystems`：EC2、FSx、CloudWatch、资源组标记 API、备份、CloudWatch 日志、KMS 和 EFS。

link:add-credentials.html["了解如何添加 AWS 凭据和设置 IAM 权限"^]。



[[link-to-fsx-for-ontap]]
=== 链接到 FSx for ONTAP

Workload Factory 需要链接才能连接到 FSx for ONTAP 文件系统。链接在 Workload Factory 与一个或多个 FSx for ONTAP 文件系统之间创建信任关系和连接。这使您能够直接通过 ONTAP REST API 调用来监控和管理某些文件系统功能，而这些功能无法通过 Amazon FSx for ONTAP API 获得。

https://docs.netapp.com/us-en/workload-fsx-ontap/create-link.html["了解如何创建链接"^]。



[[read-only-design]]
=== 只读设计

自定义仪表板为只读。它们无法更改您的存储或工作负载设置。这使您可以监控系统，而不会意外更改任何内容。



[[limitations]]
=== 限制

以下限制当前适用于自定义仪表板：

* 您不能在 Workload Factory 帐户之外共享仪表板




[[demo-environment]]
=== 演示环境

如果您使用的是 Workload Factory 演示帐户，则可以使用一些预构建的仪表板。它们可帮助您了解自定义仪表板的功能。每个仪表板都显示了一个常见的监控需求：灾难恢复状态、跨区域的容量和成本，以及是否遵守存储保护规则。



[[sample-prompts]]
=== 示例提示

以下示例展示了如何用简单的语言描述您的监控需求：

* *灾难恢复监控*："我正在 AWS 中为我的本地系统设置灾难恢复解决方案。我想查看我的本地资源的完整列表以及美国东部地区灾难恢复站点中每个资源的状态，包括其是否已配置、活跃和正常运行。"
* *多区域容量和成本*："我管理多个云区域和应用程序的存储。我希望清楚地了解已分配的存储空间量、已使用的存储空间量及其成本，以及每个区域和每个应用程序的详细信息。"
* *存储保护合规性*："我正在根据组织的标准制定新的存储保护策略。显示所有文件系统，以及它们是否满足这些要求：
+
** 数据库工作负载：每卷 12 个每日快照，每周 2 个防篡改快照。
** NFS 和 SMB 共享：每卷每天 7 个快照，每周 1 个防篡改快照。
** 必须在所有卷上启用 ARP。"
