= 对自定义信息板进行故障排除
:allow-uri-read: 
:icons: font
:imagesdir: ./media/


[role="lead"]
使用此信息可解决创建或使用自定义仪表板时可能遇到的常见问题。



[[dashboard-creation-failed]]
== 信息板创建失败

**问题：** 创建仪表板时，您会看到错误消息"无法理解请求并生成代码"。

**原因：**系统无法理解您的请求，或无法根据您的提示创建可视化效果。

**解决方案：** 尝试以下操作：

* 请使用更清晰的详细信息重写您的请求，说明您要监控的内容。
* 从 link:create-custom-dashboard.html["创建自定义仪表板"] 中的一个示例提示开始。
* 将复杂的监控需求分解为几个更简单的仪表板。
* 指定要跟踪的工作负载、指标、时间范围或维度。




[[dashboard-failed-to-load-data]]
== 仪表板加载数据失败

**问题：**已创建此信息板，但在加载数据时显示错误。

**原因：**这通常是因为权限问题或数据源不可用。

**解决方案：**

* **检查权限：** 确保您的 Workload Factory 帐户具有对仪表板使用的工作负载和资源的读取权限。查看 link:learn-custom-dashboards.html#aws-credentials-and-permissions["AWS 凭据和权限"] 中的权限要求。
* **验证凭据：** 确认您的 AWS 凭据有效且未过期。有关详细信息，请参阅 link:add-credentials.html["添加AWS凭据"]。




[[dashboard-not-loading-or-loading-slowly]]
== 仪表板未加载或加载缓慢

**问题：** 控制面板加载缓慢或某些小部件未出现。

**原因：**加载速度缓慢可能是由网络问题、大量数据或临时服务延迟造成的。

**解决方案：**

* **检查您的互联网连接：**确保您的互联网连接稳定。
* **刷新您的浏览器：** 尝试刷新此页面以查看控制面板是否正确加载。
* **清除浏览器缓存和 Cookie：** 这有助于解决由过时或损坏的数据引起的加载问题。
* **请稍等，然后重试：** 如果问题仍然存在，请稍等几分钟，然后重试。




[[missing-dashboards]]
== 缺失信息板

**问题：**您创建的仪表板不再列在仪表板目录中。

**原因：** 您的 Workload Factory 帐户中的其他用户可能已将其删除。

**解决方案：**

* 确保您已登录到正确的 Workload Factory 帐户。
* 请询问您账号中的其他用户是否有人删除了控制面板。
* 如有必要，使用原始要求重新创建仪表板。


Workload Factory 帐户中的所有用户都可以查看和删除自定义仪表板。管理共享仪表板时，请与您的团队协调。
